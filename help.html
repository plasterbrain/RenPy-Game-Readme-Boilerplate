  • Adjust line spacing scaling
  • Toggle self-voicing or clipboard-voicing mode

Self-Voicing Mode

When self-voicing mode is enabled, in-game text will be read aloud using the default speech synthesis software for your operating system. Self-voicing mode can be toggled either by pressing v, Shift + V, or through the accessibility menu.

You can use the keyboard arrows to navigate through and interact with any focusable elements on screen. When an element is focused, the game will announce the text associated with that element. When no element is focused, th game will announce whatever non-focusable text is on screen, such as the dialogue.

Self-voicing mode is not supported on Android, iOS, or Chrome OS.

Settings and troubleshooting

Windows

Self-voicing on Windows is handled by the Microsoft Speech API. Reading voice and speed can be changed by going to Control Panel, Ease of Access, Speech Recognition, and then selecting Text to Speech. This will open a window called Speech Properties where you can adjust voice settings.

Mac

Self-voicing on Mac is handled through the "say" command. Speech settings can be adjusted by going to System Preferences. Look for the following depending on your version of Mac:

  • 10.6 (Snow Leopard) and 10.7 (Lion) Speech, then the Text to Speech tab.
  • 10.8 (Mountain Lion) through 10.11 (El Capitan): Dictation & Speech, then the Text to Speech tab.
  • 10.12 (Sierra) and newer: Accessibility, then the Speech tab.
Linux

If self-voicing is not working on your Linux system, you may need to install eSpeak using your package manager.

Clipboard-Voicing Mode

If you are using a screenreader than can process text copied to the clipboard, you may prefer to use clipboard-voicing mode. Clipboard-voicing can be toggled by pressing Shift + C or through the accessibility menu.

It works in a similar fashion to self-voicing mode. If an element is focused using the keyboard, any text currently associated with that element will be copied to the clipboard. If no element is focused, any non-focusable text on screen, such as dialogue, will be copied to the clipboard instead.

Troubleshooting

Game fails to launch

If the game doesn't start, ensure your system meets the minimum requirements. If it does, the following issues may be the culprit:

Graphics drivers
In rare cases, an outdated or buggy graphics card driver will cause the game to not launch. Ensure your drivers are up to date for the best performance.
File path encoding issues
On Windows systems, the game may not start correctly if it relies on file paths containing characters outside the current system language. You will need to adjust your system locale for non-unicode programs, either through Windows settings or by using a system locale utility such as Locale Emulator.
Running on a non-primary display
On Mac OS X 10.9 (Mavericks), you will need to launch the game from the the primary display to ensure it starts correctly.

In the event you still cannot get the game to start even after following the steps in this manual, you may contact us at {support@email.com} for further instructions.

Performance issues

You may experience lag if your graphics drivers are out of date or if your system does not adequately support hardware-accelerated graphics. Please close the game and try the following troubleshooting steps:

  1. Update your graphics drivers.
  2. If playing from a laptop, you may experience better performance by keeping your computer connected to power or optimizing battery settings for performance rather than battery life.
  3. Launch the game. In the graphics settings menu (Shift + G), select a new renderer. The recommended order to try them in is OpenGL, DirectX, and then Software Renderer.
  4. Restart the game.
